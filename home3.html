  • Pneumonia and Bronchitis Detection using Cough Sounds with Audio Signal Processing

    Final Project of Information Intelligent Management Laboratory Workshop

    May 2025

    An individual project focused on detecting pneumonia and bronchitis from cough sounds using audio signal processing techniques. Conducted a comparative analysis of feature selection methods—including Sequential Forward Selection (SFS), Euclidean Distance, Linear Correlation (Mean Pearson Correlation), and Non-Linear Correlation (ANOVA and Spearman)—to extract key audio features such as MFCC, Bandwith, Contrast, and Rolloff. Utilized these features to train and evaluate machine learning models like Random Forest, Support Vector Machine (SVM), CatBoost, and XGBoost, demonstrating the impact of feature selection on improving classification accuracy for respiratory disease detection.

  • Covid-19 Cough Classification

    Final Project of KCVanguard 2025 Workshop

    March – April 2025

    Led a team of three to develop an LSTM-based deep learning model for COVID-19 detection from cough sounds using audio signal processing techniques. The project involved audio preprocessing including segmentation with Voice Activity Detection (VAD), cough validation using a pre-trained YAMNet model, and audio cleaning. Feature selection was performed using Sequential Forward Selection (SFS), resulting in key features such as MFCC, Tonnetz, Mel Spectogram, Spectral Contrast, and Chroma, which were used to train the LSTM model for classification. The project received the "Best Presentation" category.

  • WhatsApp Chat Logger Website

    Final Project of Web Programming Course

    October – December 2024

    Led a team of three to develop a web-based WhatsApp chat logger application that manages messages, contacts, and media files using a full-stack JavaScript architecture. The frontend was built with pure HTML, CSS, and vanilla JavaScript, communicating with a Node.js + Express backend via RESTful APIs. MongoDB Atlas served as the cloud database, with collections such as jsons for messages, Phonebook for contact data, and Media.files and Media.chunks for large media files stored using GridFS. The backend supported various endpoints for chat logging, media upload, and contact retrieval, ensuring real-time communication between client and server. The project emphasized clean API design, efficient data handling, and seamless integration between components.
